2. Designing Custom Moderation Workflows
a) API Gateway request validators 
- main helpful features: Rate throttling and Rate limiting 
b) Lambda Input Sanitization
- sanitize the input before passing to the Foundational model
c) Step Functions
- enables multi-stage workflows for complex content moderation
- orchestration, logic flow, resilience
d) An example of a good, secure workflow
- Input -> Step Function -> Lambda -> SageMaker Classifiers -> Bedrock Guardrails -> Foundation Model -> Post processing
- if it fails in Lambda, Sagemaker classifiers or guardrails, reroute, block or human review
e) Cloudwatch and Athena
- to collect the logs and analyze them
- this makes a good continuous improvement loop
- so capture safety events, analyze events, update controls and automate the process using Lambda

4. Reduce Hallucinations
Step 1: Using Bedrock Knowledge Bases: provides FMs and agents contextual information from trusted private data sources
  to deliver more accurate, relevant and customized responses 
  -> RAG
  -> Vector Search against trusted data
Step 2: Developing an Accuracy Verification System (evaluating how the retrieved answer aligns with the model answer)
  -> Semantic Similarity Search (checks how similar the model's answer is to the retrived source content)
  -> Retrieval Coverage (measures how well the retrieved documents cover info needed to answer a question)
  -> Confidence Scoring (combines multiple signals to estimate how reliable the model's responses is)
Step 3: Structured Output Verification (make the app return the output in a certain format)
  -> Lambda (field validation and citation checks)
  -> API Gateway (JSON schema enforcement)

5. Defense in Depth Safety Architectures (multiple layers of security controls)
Layer 1: Edge Defense (before request even reaches the system)
- WAF -> defenses against OSI Layer 7 App attacks
- API Gateway -> validates incoming requests for adherence to a predefined structure
Layer 2: Pre-Processing filters 
- Step Functions and Lamda
- Amazon Comprehend (NLP that analyzes the input text: content classification, PII detection)
Layer 3: Model Safety
- Amazon Bedrock Guardrails (use the features and filters here)
Layer 4: Foundational Model (here's your model)
Layer 5: post-processing Validation for the output before it sends back to the user
- Lambda (Custom post-processing validation for model outputs)
- API Gateway (Validate that outputs adhere to JSON schema)

Monitoring and Investigation (outside of the layers):
Cloudwatch -> EventBridge -> Lambda, Step Functions

6. Advanced Threat Detection for Foundation Models
a. Adversarial Threats
- Prompt Injection (like SQL injection but for natural language)
- Jailbreak Attempts (overwrites safety controls)
-> framing
-> authority impersonation (DAN)
-> Encoding attacks
- How to solve them? 
-> Input tagging: so we tag user input so that the guardrails only evaluate user content and not developer's instructions
-> Sagemaker Classifiers can help 
b. Adversarial Testing Workflows 
- Step Functions can help achieve this
- orchestrate: injection simulate, jailbreak attempts, red-team prompt sets
c. Detection and Automated Response
- Cloudtrail (Records who did what in AWS, such as Bedrock model invocations, API calls, changes)
- Cloudwatch (Detect anomalies such as sudden spikes in prompt volume, repeated failed moderation checks or unusual patterns in model responses)
- EventBridge (route events to automated workflows like Lambda and other Step Functions. Basically take an action here)
